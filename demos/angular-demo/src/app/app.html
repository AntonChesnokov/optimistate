Angular Signals + status/error

Advanced counter demo

Same core API, but wiring Angular signals directly for status/error and running a slower operation with a 30% failure chance.

{{ signalCounter() }}
Status: {{ signalStatus() }}
@if (signalStatus() === 'error' && signalError(); as err) {

Rolled back: {{ errorMessage(err) }}

}

Uses Angular writable signals as `WritableRef` targets for status/error, no adapters needed.

Custom rollback

Manual rollback demo

No status/error passed in; the action creates them internally. Rollback applies custom logic instead of reverting to the snapshot.

{{ customCounter() }}
Status: {{ readTarget(customAction.status) ?? 'idle' }}
@if (readTarget(customAction.status) === 'error') {

Rolled back custom: {{ customMessage() ?? errorMessage(readTarget(customAction.error)) }}

}

Custom rollback handler sets the counter to previous - 1 and records a message. Status/error are auto-created and accessible via action.status / action.error.
